# DMG Build Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Configure Tauri to produce a `.dmg` installer for macOS distribution.

**Architecture:** Add a `bundle` section to `tauri.conf.json` with DMG targets and metadata. Icons already exist. Tauri 2's built-in bundler handles DMG creation natively — no extra dependencies needed.

**Tech Stack:** Tauri 2.10.1, Cargo, npm

---

## Chunk 1: Bundle Configuration & Build

### Task 1: Add bundle configuration to tauri.conf.json

**Files:**
- Modify: `src-tauri/tauri.conf.json`

- [ ] **Step 1: Add the bundle section to tauri.conf.json**

Add the `bundle` key at the top level of the config, after `"app"`:

```json
"bundle": {
  "active": true,
  "targets": ["dmg", "app"],
  "icon": [
    "icons/32x32.png",
    "icons/128x128.png",
    "icons/128x128@2x.png",
    "icons/icon.icns",
    "icons/icon.ico"
  ],
  "category": "DeveloperTool",
  "shortDescription": "Agent Orchestrator — manage AI agent terminal sessions",
  "longDescription": "Agent Orchestrator is a desktop application for managing multiple AI agent terminal sessions with real-time status monitoring.",
  "macOS": {
    "minimumSystemVersion": "10.15",
    "dmg": {
      "appPosition": { "x": 180, "y": 170 },
      "applicationFolderPosition": { "x": 480, "y": 170 },
      "windowSize": { "width": 660, "height": 400 }
    }
  }
}
```

- [ ] **Step 2: Verify the config is valid JSON**

Run: `cd /Users/stanton.borthwick/SProjects/Agent-Orchestrator && cat src-tauri/tauri.conf.json | python3 -m json.tool > /dev/null`
Expected: No output (valid JSON)

- [ ] **Step 3: Commit the config change**

```bash
git add src-tauri/tauri.conf.json
git commit -m "feat: add DMG bundle configuration to tauri.conf.json"
```

### Task 2: Build the DMG

- [ ] **Step 4: Run the Tauri build**

Run: `cd /Users/stanton.borthwick/SProjects/Agent-Orchestrator && npm run tauri build`

This will:
1. Run `npm run build` (TypeScript + Vite frontend build)
2. Compile the Rust backend in release mode
3. Bundle into `.app` and `.dmg`

Expected output location: `src-tauri/target/release/bundle/dmg/Agent Orchestrator_0.1.0_aarch64.dmg` (or `x86_64` depending on architecture)

- [ ] **Step 5: Verify the DMG was created**

Run: `ls -lh "src-tauri/target/release/bundle/dmg/"`
Expected: A `.dmg` file listed

- [ ] **Step 6: Smoke-test the DMG**

Run: `open "src-tauri/target/release/bundle/dmg/"`

Manual verification:
1. Double-click the `.dmg` to mount it
2. Verify the app icon and Applications folder shortcut appear
3. Drag to Applications
4. Launch from Applications and confirm the app starts

- [ ] **Step 7: Commit any build-related changes if needed**

```bash
git add -A
git commit -m "chore: verify DMG build succeeds"
```
